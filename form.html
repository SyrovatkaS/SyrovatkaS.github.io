Тег Изображение и его атрибуты

Изображение вставляется в текст с помощью тега img и атрибута src (сорс - источник), указывающего на файл. Также можно атрибутами можно задать высоту height и ширину  width изображения. Если задать только одно из этих значений, пропорция фотографии будет сохранена. Поэтому надежнее задавать только что-то одно.

Кроме того, можно универсальным атрибутом align задать обтекание текстом изображения справа или слева. Атрибут alt позволят прочитть, что есть на изображении, если картинки в браузере заблокированы.

Также можно задать отступы от изображения до краев страницы и текста. Отступ по горизонтали задается атрибутом hspace, а по вертикали  vspace.

Тег Жирный шрифт и курсив

В жирный шрифт текст преобразовывается тегом strong. Это парный тег, который выделяет именно ту область текста, которую нужно. Но все это нужно делать в CSS.

Курсив создается в тексте тегом em. Он тоже парный, и тоже с двух сторон выделяет ту область текста, которая должны быть курсивом.Но все это нужно делать в CSS.

Формы

Тег Формы используется для создания различных полей, которые нужно заполнить, после чего информация будет отправлена, куда укажем. Нпример, это может быть форма сообщения авторам сайта.

При формировании формы нужно указать для нее действие action, то есть програмный блок, созданный на языке программирования - в данном случае язык php. После этого нужно указать метод применения формы, чаще всего это метод post, и имя формы тоже указать обязательно, потому что форм может быть несколько.

Для создания формы для ввода используется тег input (переводится ввод), и в этом теге сразу указывается типа ввода type. Чаще всего вводится текст, значит применяет атрибут text. Затем нужно ввести имя для строки ввода. В нашем случе в это строку нужно будет вводить имя пользователя name.

Далее создает описания формы ввода. Это можно сделать несколькими вариантами. Самое просто: создаем абзац перед тегом input, и пишем описание: Введи имя.

Чтобы форма ввода сразу активировалась при щелчке мышкой не только на саму форму, но и на текст описания, нужно создать парный тег Lable (этикетка)

Далее атрибутом size можно подобрать подходящую ширину поля ввода. Ее можно указать в символах, просто написав цифру, или в пикселях, прочто написав цифру с px. Автоматически поле ввода имеет ширину в 20 символов. Но обычно ширина поляввода указывается в CSS.

Также можем ограничить максимальное количество символов в поле ввода.

Абсолютно так же создается и поле для ввода пароля. Только в типа тега нужно применить атрибут не text, а password. В этом типе ввода мы не будем видеть, что вводим, будут показаны только круглые маркеры.

Радиокнопки

Радиокнопки используются для опросников. Самые простой опросник - да или нет. Создается он как и форма ввода текста, но с типом не text, а radio.

Кнопка Да создается с помощью атрибута value yes, кнопка нет, соответственно value no. Атрибут Name time даст возможность выбрать только один из ответов. Без этого атрибута будут доступны оба ответа одновременно. Для создания альтенативной кнопки просто копируем всю строку, начиная с labelЮ и меняем да на нет.  

Сам вопрос для ответа да или нет можно написать сразу после тега Абзац. Приэтом удобно, если вопрос написан в отдельном обзаце, тогда нопки будут под ним.

Если нужно, чтобы какая-то из кнопок выбора была активирована еще до ответа, нужно дописать к соответствующему ответа атрибут checked

Для каждой кнопки важно дать отдельное имя. Это потом поможет правильно обработать ответы из формы ввода, который будут переданы автору опроса.

Select - выбор из выпадающего списка.

Для создания опросника с выбором ответом из выпадающего окна используется тег select. Код очень поход на тег Список. Варианты ответов выделяются парным тегом option (выбор)

Блокировка и Только чтение

Блокировка и Только чтение - два тега, которые, позволяют соответственно остановить применение части формы или всю ее, и запретить любиые изменения в форме - режим только чтение.

Блокировка применяется с помощью тега desabled (неактивно). Он вставляется внутри тега формы, которую мы хотим сделать неактивной

Карта изображения

Иногда на сайте необходимо некоторые его области или элементы сделать ссылками. Для этого нужно создать карту изображения, задать координаты области, которая должна стать кликабельной, и сам линк, на который переадресует клик.

Сначала вставляем в код картинку, используя тег img src ="", указывем размер по ширине и высоте, альтернативную подпись, и вертикальный или горизонтальный отступ (если нужно). Затем просто в начале кода картинки прописываем тег ссылки a href="", указываем, куда она должна вести, не забываем указать открывание в новом окне атрибутом target=", и закрываем тег a за кодом картинки. Будильник стал линкабельным.

Но иногда возникает необходимость, чтобы разные части одного рисунка были кликабельны и вели на разные сайты.

Для этого стандартно загружаем в html файл, и в конце кода картинги прописываем тег usemap="", с указанием на идентификатор области изображения или все изображение, которое станет линком.

Сам же линк на изображении создается строкой ниже, тегом map с указанием идентификатора mapID="". Внутри тега map прописывается атрибут area.

Для атрибута area прописываем три главных характеристики: 1) куда будет вести ссылка href=", 2)форму активного поля shape="rect" (rectangle - прямоугольник), "" (poligon - многоугольник), "cicle" (круг), 3) coords - координаты, указывающие на удаленность необходимой области начала и концов изображения от левого края страницы и от верха. Автоматически координаты выделенной области можно получить на сайте http://maschek.hu/imagemap/imgmap/. Также для этого используются специальные программы, некоторые из которых платные.

Сделав разные линки для двух половин изображения, получаем возможность переходить на соответствующие сайты по продаже настольных или песочных часов.

Тут же тег title позволяет сделать вспрывающую подсказку на каждой из половин изображения.

А тегом target, как всегда при использовании линка, не забываем указывать, чтобы ссылка открывалась в новом окне.

Базовый курс CSS

CSS - язык разметки страницы, отвечающий за стили. Он взаимодействует с файлом HTML для создания гармоничного вида сайта.

Сначала создаем файл CSS в блокноте. Это обычный файл с расширением css. Затем привязываем его к файлу. Это делается созданием соответствующей строки в теге head.

Строка начинается с тега link, в котором атрибут href тут же указывает папку и название файла  css: link href=". Затем следует атрибуты типа type и взаимоотношения файлов rel, которые всегда неизменны. После появления этой строки файлы html и css связаны. К одному файлу scc может быть привязано любое количество файлов css, и наоборот.

Три базовых селектора CSS

Три базовых селектора  CSS, которые применяются в 90% случаев: 1) селектор по тегу, 2) селектор по идентификатору, 3) селектор по классу.

Селектор по тегу позволяет применить общее правило для всех тегов одного типа, которые встречаются в файле HTML. Например, p {color=#значение цвета в формате Hex RGB}. В нашем случае мы применили изменение цвета всего содержимого в теге p.

Селектор по идентификатору позволяет применить правило к одному определенному тегу. Для этого сначала в файле CSS прописываем правило, а затем в любом теге его применяем. На примере этого абзаца показано, как изменить цвет текста только внутри одного тега.

Третий базовый вид селекторов используется, если нам нужно присвоить правило не одному тегу, а многим, или даже отрывку текста. Можно было бы создавать каждый раз новый идентификатор, но удобнее и правильнее использовать для этого селектор по классу  class. Для этого в файле CSS сначала формируем класс .new {color:#}, а затем применяем его в разных частых текста HTML в виде class="new".

Это позволяет в нашем файле сделать несколько абзацев бирюзовым цветом шрифта.

Если нужно применить свойство класса к отрывку предложения, помещаем класс в парный тег span.

Групповые селекторы

Если существует необходимость применить общее правилок абсолюто разнотипным элементам, можно под каждое использовать или идентификатор, или под каждый тег, но это не очень продуктивно, и делает код слишком громоздким. Рациональнее использовать групповой селектор.

Для реализации группового селектора просто через запятую пишутся разные элементы, а затем для них прописывается общее правило.

Например: h3, p, option {color:987168;}

Селекторы потомков

Теги делятся на предков и потомков в зависимости от своей взаимосвязи. Например, все теги a, p, h являются потомками по отношению к тегу body.

Селектор потомков позволяет задать общее свойство для указанного потомка определенного тега.

Например, ol li {color: #...} - это означает, что определенный цвет задан для всех пунктов цифровых списков.

Псевдоселекторы ссылок

Псевдоселекторы ссылок позволяют с помощью CSS изменять цвета ссылок в четырех состояниях: 1) когда ссылка в стационарном состоянии, 2) когда на ссылку уже кликнули, 3) когда на ссылку навели мышью, 4) момент, когда на ссылку кликают.

Семейство шрифтов

Шрифт можно менять как для всего сайта, так и для любой из его частей. Для этого в таблице стилей прописывается сначала область, к которой нужно применить шрифт, потом свойство font-family, и потом название шрифта.

Например: h {font-family: "Verdana";}

Название нужно взять в скобки. Можно указывать несколько шрифтов, тогда браузер пользователя будет подбирать приоритетный шрифт именно в указанной последовательности: h {font-family: "Verdana", "Times New Roman";}

При выборе шрифтов важно отбирать такие, которые одновременно поддерживаются операционными системами Windows и iOS.

Для надежности после перечня приоритетнх шрифтов, можно указывать один из классов. Тогда, если операционная система не найдет реконммендуемые шрифты, она просто возьмет шрифт из указанного класса шрифтов. Таких классов самых распространенных только два: serif - с засечками, и без засечек - sans-serif: h {font-family: "Verdana", "Times New Roman", serif;}

Размер шрифта

Сразу после шрифта обычно в CSS строкой ниже указывают размер шрифта, особенно если он должен отличаться от базового. Базовым является размер шрифта 16 px. В таблице стилей размер шрифта указывается очень похоже на название шрифта.

 Например: h {font-size:20px;}

Также размер шрифта можно указать в процентах (16px=100%), в em - это измерение пришло из книгопечатания, и 1 em=16px. Ткже могут измерять в пунктах и клучевых словах midium large smal, но это встречается крайне редко.

Жирность шрифта и курсив

Жирность и курсив для шрифта реализуется с помощью селектора классов, и двух соответствующих свойств - font-weight и font-style.

Сначала создаем класс, который будем по всему тексту применять для жирного шрифта - удобно назвать класс wFont (от weight font). Затем в этом классе прописываем свойство font-weight, и его значение bolt. Нежирный шрифт имеет значение normal. Жирность шрифтов также может иметь цифровое значение, в котором 100 - нежирный шрифт, 400 - полужирный шрифт, максимальное значение - 900. Но не все браузеры цифровое значение поддерживают. Если нужно сделать только часть текста, выделяем его парным тегом span.

Курсив создается также с помощью селектора класса (удобно использовать название класса iFont), а также свойства font-style, и его значения italic.

Высота строки, интервал между словами

Высота строки обычно равна размеру шрифта. Но мы можем менять это значение свойством line-height. Значение можем задать в пикселях, но можно и просто применить множитель. Если значение будет, например, 2, то высота строки увеличивается в два раза. Можно применять и проценты. Это удобно, если нужно подогнать текст к определенных размерам на странице - так делают верстальщики.

Для этих же целей можно менять интервал между словами. За это отвечает свойство word-spacing. Здесь проценты не используют, только пиксели. Важно знать, что если для абзаца задано свойство justify, вручную расстояние между словами отрегулировать не получится.

Также для регулирования расстояния между буквами можно использовать свойство letter-spacing. Это может быть актуально для заголовков, которые состоят из нескольких букв, и визуально интересно их растянуть или сузить.

Регистр в тексте меняется свойством text-transform. Все буквы заглавными, например в заголовке, можно отобразить, задав значение text-transform="uppercase". Если нужны все маленькие - lowercase, если нужны с большими буквами в начале каждого слова - capitalize.

В принципе, можно было бы все эти свойства просто написать как хотим в HTML, но обязая идеология предусматривает любые свойства, связанные с внешним видом, прописывать в CSS.

Сокращенная запись свойств шрифта

Чтобы код не выглядел перегруженным, используют сокращенную запись, указывающую на 1) курсив, 2) жирность, 3) размер и 4) шрифт текста. В таблице стилей для класса font прописываем ссвойства: эти значения через пробел именно в такой последовательности, и тогда браузер распознает все необходимые установки. Применим это к текущему обзацу, прописав класс sokr и его свойства: italic bold 16px "times new roman".

Через слеш / после указания размера шрифта можно тут же оказать межстрочное пространство в пикселях. Но необязательно. Указывает курсив и жирность тоже не обязательно - в таком случае будет автоматически применен стиль без курсива и без жирности.

Рамка

Рамка используется на сайтах достаточно часто. За рамку в таблице стилей отвечает элемент border. Три его главные свойства: толщина, стиль и цвет. Их значение можно указать сокращенно для элемента border именно в этой последовательности. Но можно и отдельно как свойство какого-то тега, идентификатора или класса в виде значений border-weight (в пикселях), border-style (solid если сплошная рамка, dashed - в виде тире, dotted - точками, double - двойная), и border-color.

Удобнее конечно указывать свойства рамки в сокращенном виде border: 2 solid #333333

Можно выводить только часть рамки, применив элементы border-top (верх), border-bottom (низ), border-left и botom-right. В частности, border-bottom можно использовать для определения ссылок нижнего подчеркивания ссылок. При этом также в сокращенном виде можно тут же задать свойство этого элемента.

Если к одному элементу напрямую применены свойства разными способами, то вступает в силу правило значимости стилей: селектор тегов - 1 балл, селектор классов - 10 баллов,  ID-селектор - 100 баллов.

Наследование каскадность и приоритетность

Принцип наследования позволяет сэкономить время при написании кода. Свойства, заданные для элементов-предков, наследуются элементами-потомками. Если мы для таблице стилей для body зададим размер шрифта, его семью и цвет, то все элементы внутри тега body примут соответствующий вид.

Не наследуют свойства у тегов-предков заголовки, ссылки (берут шрифт и размер, но не цвет) и рамки. Эти свойства встроены в стилях браузера,и не меняются. Свойства, которые не наслудуются, нужно прописывать напрямую.

Принцип каскадности - это учитывание одним тегом разных свойств, заданых в в разных элементах таблицы стиля.

Первый принцип приоритетности: если элемент наследует свойство от двух и более родителей, он учитывает свойство, задаваемое ближайщим родителем.

Второй принцип приоритетности: если для элемента непосредственно заданы конкретные правила, все наследования им игнорируются.

Блочная модель. Padding, margin.

Каждый тег для браузера имеет блочную структуру, которую можно свободно менять. Для важно понимать, что каждый блок для CSS делится на содержимое, внутренний отступ - padding, рамку - border, внешнее поле или внешний отступ - margin. К каждому из этих элементов добавляются свойства top - верх, bottom - низ, left и right. Как и для таблицы, блоку можно делать отступы с одной из сторон.

Эти свойства не наследуются. То есть,если здали для body, абзацы не наслудуют. Единица измерений - в пикселях.

Для короткой записи разных значений для верха, низа, правой и левой сторон, можно использовать через пробел значения в последовательсти от top и по часовой стрлке. Например, pedding: 40 20 30 35;Если верхнее и нижнее значения одинаковы, и правые и левые одинаковы, то достаточно задать тольо два значения, сначала верх-низ, и блаузер этой поймет.

Ширина и высота блока

Если не указаны свойства ширины и высоты блока, то браузер отображает блок с высотой шрифта, и шириной на всю ширину страницы.

За ширину блока отвечает уже знакомое свойство width. Значение устанавливается в пикселях. Можно и в процентах, тогда за 100% будет считаться ширина родителя. Очень важно помнить, что в данном случае мы задаем лишь ширину содержимого, без учета ширины рамки (border), внутреннего (padding) и внешнего (margin) отступов - они занимают отдельное пространство.

Выcота блока задается с помощью свойства height

Если контент не помещается в блок, для него задается правило, которое определяется свойством overflow (переполнение). Значения для этого свойства могут быть такими:

  • hidden - спрятать контент, который не помещается;
  • auto - будет добавлена полоса прокрутки, чтобы показать ту часть, которая не помещается;
  • scroll - добавляет полосу проекрутки принудительно, даже если контент помещается в блок.
    •

Для выравнивания блока в свойство margin, отвечающее за вынешний отступ, задаем значения righ и left в виде auto. В сокращенном написании это может выглядеть margin: 40px auto 20px auto.

Конфликт полей

Если два блока сталкиваются размерами полей, то браузер берет большее значение одного из блоков, а меньшее игнорируется. То есть значения отступов двух элементов не складываются, а берется большее значение.

Такое же правило действует и в случае конфликта вложенного элемента с родительским. Конфликт полей брайузер решает, учитывая большее значение.

Блочные и встроенные элементы

Вся страницасостоит из блочных элементов. Внутри себя он может содержать встроенные элементы, например с тегом span.

Встроенные элементы не могут содержать в себе блочные элементы.

У встроенных \лементов нет понятия верха и низа, соответственно, нет свойств top и bottom, только свойства right и left.

Преобразовать блочный элемент во встроенный и наоборот можно с помощью тега display. Из встроенного в блочный преобразуем с помощью CSS, например ol li {display: input;}

Наоборот, из встроенного в блочный преобразовуем так: ol li {display: block;

Для удобства верстки блоков удобно задавать контур для элемента с помощью свойства outline, для которого значеия задаются как и для таблицы, но используются они исключительно для верстки.

Фоновый цвет и изображение

Цвет фона любого элемента задается свойством backgraund-color. Изначально цвет фона задан как прозрачный для всех элементов.

Цвет фона можем задать для тега body, и это преобразует фон всей страницы. Таким же образом можем задать фон и для заголовка - через тег header.

Цвет фона задается для содержимого блока и внутреннего отступа padding. Рамка border может быть другого цвета. Внешний отступ margin никак не раскрашивается, это просто отступ от элемента.

С помощью тега div можем создать отдельные блок, которым зададим оригинальный фон.

Чтобы сделать фотографию фоном, в CSS используем свойство background-image, и указываем путь к фото через url ().

Повторение фонового изображения

Для ускорения работы сайта фоновое изображение можно загружать не полностью, а только маленьким элементом. При этом сначала в backgroud-image: url() загружаем патерн (маленькое изображение), а потом с помощью элемента background-repeat:repeat мы просто тиражируем это изображение на всю страницу.

При необходимости можно задать тиражирование только по горизонтали (оси х), или вертикали (оси у). Для этого указываем background-repeat:repeat-x или background-repeat:repeat-y.

Позиционирование фонового изображения

Для того, чтобы рядом с текстом (в нашем случве это заголовок сайта), который расположен на фоне, разместить небольшую картинку, используем background-image, и в url указываем путь к нему. Но если больше ничего не сделать, изображение просто станет фоном на всю площадь элемента. Поэтому с помощью background-size указываем размер фонового изображения. Также в теге background-repead ставим значение no-repeat. Таким образом получаем картинку нужного размера.

Далее с помощью тега background-position указываем в пикселях отступ по горизонтали и вертикали от првого верхнего угла (это позиционирование имеем по умолчанию).

Короткая запись для фона

При записи свойств элемента background получается достаточно громоздкий код. Для упрощения используется короткая запись свойств.

Короткая запись для фона осуществляется в таком порядке:

  1. background-color
  2. background-image
  3. background-repeat
  4. background-attachment
  5. background-position

Если фоновых изображений несколько, каждое из них можно записывать в тег background через запятую (тольков CSS3). Также можно использовать свойство background-size для размера фонового рисунка. Кроме того, CSS3 понимает произвольную очередность свойств при короткой записи для фона.

Фиксация фонового изображения на прокрутке

При прокрутке страницы фоновое изображение может оставаться видным. Для этого нужно использовать свойство background-attachment:fixed.

С помощью background-position в пикселях назначаем расположение фоновой картинки по вертикали и горизонтали относительно верхнего левого угла.

Свойство float

Свойство Флот позволяет создать, так называемые, плавающие элементы, которые выдны на странице поверх других, и обтекаются ими.

Таким образом мы задаем обтекание не в html, как раньше, а в таблице стилей.

В переводе float - обтекающий. Это созвучно по смыслу со словом флот.

Задаем идентификатор для картинки будильника, и дальше прописываем ему свойства в таблице стилей.

Свойство float имеет значение вправо или влево, и передвинув будильник вправо, получае автоматическое его обтекание текстом.

Очень важно учитывать, что обтекание осуществляет только контент блоков абзаца. А сами абзацы сохраняют ширину блока на всю ширину страницы. То есть, ширина контейнеров не меняется. Это понимание пригодится при блочной верстке сайтов.

Фактически свойство float приподнимает изображение над другими элементами, которые занимают ее место, но невидны. При этом контент блоков будильник обтекает.

Обтекает картинку тот текст, который находится ниже тега img,в котором находится картинка.

Чаще всего эти свойства используются при верстве для блоков div.

Очистка обтекания float

При использовании тега float важно предварительно задать блоку ширину. Только в таком случае блок, который расположен ниже, сможет произвести правильное обтекание. Иначе он просто не сможет зайти на территорию блока, который занимют всю ширину сайта.

Создадим второй сайдбар, и расположим его справа от блока контента.

Уменьшаем ширину блока контент, и прописываем ему обтекание через margin с обеих сторон auto. В итоге получили каркас сайта с двумя сайдбарами.

Теперь заводим контент в сайдбары. Чтобы контент не выходил за пределы блока, создаем класс hidden с правилом overflow:hidden;

После наполнения сайдбаров контентом блок Футер плотную поднялся к локу контент, но нам это не нужно. Как отменить требование обтекания для конкретного блока. Для этого используется очистка обтекания.

Это реализуется с созданием в html пустого div перед блоком, для которого нужна отмена обтекания. Внутри пустого дива прописываем класс clear, а в таблице стилей внутри коротого прописывается этот класс, и правило clear со значением отмены обтекания справа, слева, или с обеих сторон: right, left, both.

Отмена действует для всех блоков, которые находятся ниже дива с класом клир. В итоге футер сразу опустился вниз.
